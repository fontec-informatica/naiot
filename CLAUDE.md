# CLAUDE.md — Instruções do Projeto NAIOT

## Idioma

Sempre responder em **português do Brasil** em todas as interações.

## Git e Deploy

- Fazer **commit e push no GitHub antes e depois de cada alteração** no código.
- Mensagens de commit devem ser claras e em português.
- Após o push, verificar se o deploy via FTP foi concluído com sucesso.

## Validação

- Sempre validar o resultado após qualquer alteração (HTML válido, links funcionando, layout correto).
- Antes de reportar uma tarefa como concluída, confirmar que as mudanças estão refletidas no servidor.

## Repositório

- GitHub: https://github.com/fontec-informatica/naiot
- Servidor: 192.185.222.200 — /public_html

## Fluxo de trabalho

1. Commit e push antes de iniciar alterações (registrar estado anterior)
2. Realizar as alterações
3. Validar localmente
4. Commit e push das alterações
5. Confirmar que o FTP sincronizou com o servidor
